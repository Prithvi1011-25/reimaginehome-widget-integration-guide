Enterprise Integration Guide

A drop-in design tool for any property listing.

The ReimagineHome widget is a self-contained interface partners embed in their app to let users restyle the rooms in a listing. It loads on demand as an overlay, talks directly to ReimagineHome's servers, and returns when the user is done. Two integration paths. One configuration model.

Quick start How it works Framework guides

Quick start

Five steps to a working widget. Pick CDN if you want zero install; pick npm if you're in a React, Next, Vue, or Angular app. The shape of the steps is the same either way.

  1. Request a publishable key

    Email partners@styldod.com with your partner name and the domains the widget will load on. We'll provision a publishable key scoped to those origins — replace the public_key placeholder in your integration config.

  2. Load the widget

    Either drop the script tag on your page or install the package.

    CDN 
    <script async
  src="https://cdn.jsdelivr.net/npm/reimaginehome-widget/dist/widget.js"
  data-public-key="public_key"></script>
    npm 
    npm install reimaginehome-widget

  3. Configure it once on page load

    Pass the publishable key and any persistent defaults — branding, theme, language. The widget keeps this until you reconfigure or destroy it.

  4. Open it on user action

    Call await reihWidget.open({ media, mode }) with the media items you want the user to redesign. Per-launch config overrides are allowed here.

  5. Close or destroy when done

    Call reihWidget.close() to hide the modal while keeping the session warm, or reihWidget.destroy() to tear it down completely. Always destroy on route change in SPAs.

Skip ahead to the framework guides for the exact code your stack needs.

How the widget works

The package you install is a small loader. When you call open(), the loader creates a session with the ReimagineHome backend and mounts a hosted iframe app over your page. The heavy UI runs in the iframe, which means updates roll out without you having to republish anything. Your app's role is to deliver the loader, configure it once, and invoke it on demand.

Both integration paths converge on the same loader. The loader creates a session and mounts the hosted iframe app.

The three responsibilities

Regardless of which integration path you take, your application has three responsibilities and the widget handles everything else.

  1. Deliver the loader. Either include the script tag (CDN) or install and import the package (npm). Both produce the same handle to interact with.
  2. Configure once. Pass your publishable key and any persistent defaults at app boot. The loader retains this configuration for every subsequent invocation.
  3. Invoke on demand. When a user action should launch the widget, call open() with the per-launch payload — typically the media items and any overrides. The loader handles session creation and mounting from there.

Lifecycle of an invocation

  1. Your application calls open() with the payload for this launch.
  2. The loader requests a session from the ReimagineHome backend.
  3. The loader mounts a hosted iframe app as a fixed-position overlay above your page and sends configuration via postMessage once the app signals ready.
  4. The iframe app runs the full UI and communicates with the backend as needed.
  5. When the user closes the widget, the overlay unmounts and focus returns to your page.

Your page underneath is never navigated away from or mutated. The widget owns its own iframe, its own styles, and its own state. Closing it returns the user to exactly where they were.

Integration requirements

Three things to have lined up before you write integration code. Skipping any of these is the most common reason an integration looks fine in dev but breaks in production.

A publishable key, scoped to your domains

The widget authenticates each session with a publishable key you set as public_key in configure() or on the script tag's data-public-key attribute. The key is bound to a list of allowed origins — requests from other origins are rejected. You can have separate keys for staging and production.

The key is safe to ship in client code. It's not a secret, but it is enforced against your origin list, so leaked keys can't be used from elsewhere.

Publicly reachable media URLs

The widget passes each media[].image_url to ReimagineHome's backend, which then fetches the images server-side. This means every URL you pass to open() must be reachable from the public internet — not from your CDN behind a VPN, not from localhost, and not bundled inside your app build.

Common pitfall. A bundler-resolved path like /src/assets/listings/01.webp works in your UI because your bundler resolves it. But that path is meaningless to ReimagineHome's backend. You need a parallel public URL — e.g. https://yoursite.com/listings/01.webp — and that's what you pass as image_url.

Local development

On localhost, browsers block direct calls to ReimagineHome's API on CORS grounds. Run a thin dev proxy that forwards /api/reih/* to the ReimagineHome API and you'll match production behavior. Most React, Next, and Vite dev servers support this in a few lines of config.

A trigger that calls open()

The widget doesn't insert buttons for you. You decide where the trigger appears in your UI and wire each one to a call to await reihWidget.open({ media, mode }). You can pass a single media item or many — the widget handles both.

Configuration reference

Persistent configuration applied to every open() call until you reconfigure or destroy. Set via reihWidget.configure() (npm) or window.reihWidgetConfig + data-public-key (CDN).

CDN exception. For CDN integrations, public_key is set on the script tag's data-public-key attribute, not inside window.reihWidgetConfig. All other fields below go in the config object.

Key Type Required Description
public_key string Required Your publishable key. Set as public_key in configure options or data-public-key on the script tag. Bound to your allowed origins.
mode "simple" | "agentic" Optional Widget flow mode. Defaults to "simple". The "agentic" mode is available; documentation forthcoming.
branding WidgetBranding Optional Logo, heading, colors, footer. See Styling & branding for the sub-fields.
theme WidgetTheme Optional Colors, font, radius — visual tokens applied to the widget UI.
language WidgetLanguage[] Optional Languages available in the in-widget switcher. Each entry: { code, name, nativeName }.
sidebar_position "left" | "right" Optional Which side of the overlay the sidebar mounts on.
user_id string Optional Your own analytics identifier for the end user. Echoed in events.
session_id string Optional Your own reference id for this user's flow. Distinct from the widget's internal session.

language entries

Each entry in the language array is an object describing one selectable UI language.

WidgetLanguage 
language: [
  { code: "en-US", name: "English (United States)", nativeName: "English (US)" },
  { code: "en-GB", name: "English (United Kingdom)", nativeName: "English (UK)" },
  { code: "es-ES", name: "Spanish", nativeName: "Español" },
]

The open() payload

open() is async and returns a Promise. The argument is per-launch — media is required, everything else overrides the persistent config for this launch only.

Key Type Required Description
media MediaItem[] Required Non-empty array. Each item is { image_url: string }. Pass one for single-media flow, many for batch.
mode "simple" | "agentic" Optional Override the configured mode for this launch.
branding WidgetBranding Optional Override the persistent branding for this launch only. Useful for multi-brand portals.
theme WidgetTheme Optional Override the persistent theme for this launch.
language WidgetLanguage[] Optional Override the available languages for this launch.
sidebar_position "left" | "right" Optional Override the sidebar position for this launch.
onError (err) => void Optional Called with { message, code? } when something goes wrong during the session.
onClose () => void Optional Called when the modal closes.

MediaItem

Each media entry is an object with at minimum an image_url pointing to a publicly reachable image.

MediaItem 
media: [
  { image_url: "https://yoursite.com/listings/01.webp" },
  { image_url: "https://yoursite.com/listings/02.webp" },
]

Other controls

Two helpers cover the session lifecycle after open().

  • reihWidget.close() — hide the modal, keep the underlying session warm. Reopening is fast.
  • reihWidget.destroy() — tear down completely. Use on route change in SPAs, and before reopening with a different configuration.

2. Shared widget config

Both npm and CDN paths share one module for media, branding, language, and open helpers. The npm path puts public_key inside configure().

src/widgetConfig.ts
export type ReihMediaItem = { image_url: string };

export const WIDGET_PUBLIC_KEY = 'public_key'; // replace with your key

export const LISTING_MEDIA: ReihMediaItem[] = [
  { image_url: 'https://yoursite.com/listings/01.webp' },
  { image_url: '/images/apartment-building.png' }, // resolved to absolute URL
];

/** Local /images/* paths need a full URL for the widget backend */
export function resolveMediaUrl(url: string): string {
  if (url.startsWith('/') && typeof window !== 'undefined') {
    return `${window.location.origin}${url}`;
  }
  return url;
}

export function resolveListingMedia(media = LISTING_MEDIA) {
  return media.map((item) => ({
    ...item,
    image_url: resolveMediaUrl(item.image_url),
  }));
}

export function buildWidgetBranding() {
  return {
    logo: 'https://ecdn.styldod.com/assets/logo/6a2bca9bce2a355c2c13d058.svg',
    colors: {
      primary: '#071121FF',
      textPrimary: '#071121FF',
      textSecondary: '#1B232E',
    },
    heading: 'Reimagine Your Space',
    subHeading: 'AI-powered room redesign',
    footerText: '',
  };
}

export function buildWidgetLanguage() {
  return [
    { code: 'en-US', name: 'English (United States)', nativeName: 'English (US)' },
    { code: 'en-GB', name: 'English (United Kingdom)', nativeName: 'English (UK)' },
    { code: 'pl-PL', name: 'Polish', nativeName: 'Polski' },
    { code: 'es-ES', name: 'Spanish', nativeName: 'Español' },
  ];
}

export function clearReihLoader() {
  document.getElementById('reih-host-loader')?.remove();
}

export async function openReihWithMedia(widget, media: ReihMediaItem[]) {
  clearReihLoader();
  await widget.open({
    media: media.map((item) => ({
      ...item,
      image_url: resolveMediaUrl(item.image_url),
    })),
    mode: 'simple',
    branding: buildWidgetBranding(),
    sidebar_position: 'right',
    language: buildWidgetLanguage(),
  });
}

export function buildNpmWidgetConfigureOptions() {
  return {
    public_key: WIDGET_PUBLIC_KEY,
    media: resolveListingMedia(),
    mode: 'simple' as const,
    branding: buildWidgetBranding(),
    sidebar_position: 'right' as const,
    language: buildWidgetLanguage(),
    onComplete: (detail) => console.log('[reih] onComplete:', detail),
    onError: (err) => console.error('[reih] onError:', err),
    onClose: () => console.log('[reih] onClose'),
  };
}

Media shape. Each item is a flat { image_url: string }. Use the same URL for your <img> and the widget API — resolve local paths before passing them to open().

3. Vite dev plugin (optional)

The npm package ships with production URLs baked in. If you need local dev to hit the same backend as your CDN embed, add a transform plugin:

vite.config.ts
import { WIDGET_DEV_API_BASE_URL, WIDGET_DEV_APP_URL,
         WIDGET_PROD_API_BASE_URL, WIDGET_PROD_APP_URL } from './src/widgetEnv';

function reihWidgetDevEnv() {
  return {
    name: 'reih-widget-dev-env',
    transform(code, id) {
      if (!id.includes('node_modules/reimaginehome-widget/dist')) return;
      return {
        code: code
          .replaceAll(WIDGET_PROD_API_BASE_URL, WIDGET_DEV_API_BASE_URL)
          .replaceAll(WIDGET_PROD_APP_URL, WIDGET_DEV_APP_URL)
          .replace('return "prod"', 'return "dev"'),
      };
    },
  };
}

4. Configure and open from a page component

Call configure() once on mount, destroy() on unmount. No pre-open destroy for npm — just guard double-clicks and call the shared open helper.

src/pages/PackageIntegrationPage.tsx
import { useCallback, useEffect, useRef } from 'react';
import { reihWidget } from 'reimaginehome-widget';
import {
  buildNpmWidgetConfigureOptions,
  clearReihLoader,
  openReihWithMedia,
  resolveListingMedia,
  type ReihMediaItem,
} from '../widgetConfig';

export function PackageIntegrationPage() {
  const openingRef = useRef(false);

  useEffect(() => {
    reihWidget.configure(buildNpmWidgetConfigureOptions());
    return () => {
      reihWidget.destroy();
      clearReihLoader();
    };
  }, []);

  const openWidget = useCallback(async (media: ReihMediaItem[]) => {
    if (openingRef.current) return;
    openingRef.current = true;
    try {
      await openReihWithMedia(reihWidget, media);
    } catch (error) {
      clearReihLoader();
      console.error('[package] Widget open failed:', error);
    } finally {
      openingRef.current = false;
    }
  }, []);

  const handleOpenAll = useCallback(() => {
    void openWidget(resolveListingMedia());
  }, [openWidget]);

  const handleOpenSingle = useCallback(
    (media: ReihMediaItem) => { void openWidget([media]); },
    [openWidget],
  );

  return (
    <ListingDemoPage
      onOpenAll={handleOpenAll}
      onOpenSingle={handleOpenSingle}
    />
  );
}

This integration injects widget.js dynamically from a React page — not from index.html. Useful when you want the script on a specific route without adding the npm package to your bundle.

1. Load the script from a page component

Set window.reihWidgetConfig before the script loads. Put public_key on the script tag's data-public-key attribute, not in the config object.

src/pages/ScriptEmbedPage.tsx
import { useCallback, useEffect, useRef } from 'react';
import {
  WIDGET_PUBLIC_KEY,
  WIDGET_SCRIPT_URL,
  buildScriptEmbedWidgetConfig,
  clearReihLoader,
  openReihWithMedia,
  resolveListingMedia,
  waitForReihWidget,
  type ReihMediaItem,
} from '../widgetConfig';

const WIDGET_SCRIPT_ID = 'reih-widget-script';

function setScriptEmbedConfig() {
  window.reihWidgetConfig = buildScriptEmbedWidgetConfig();
}

export function ScriptEmbedPage() {
  const openingRef = useRef(false);

  useEffect(() => {
    setScriptEmbedConfig();

    let script = document.getElementById(WIDGET_SCRIPT_ID) as HTMLScriptElement | null;

    if (!script) {
      script = document.createElement('script');
      script.id = WIDGET_SCRIPT_ID;
      script.src = `${WIDGET_SCRIPT_URL}?v=${Date.now()}`;
      script.async = true;
      script.setAttribute('data-public-key', WIDGET_PUBLIC_KEY);
      script.addEventListener('load', setScriptEmbedConfig);
      document.body.appendChild(script);
    }

    return () => {
      window.reihWidget?.destroy?.();
      clearReihLoader();
    };
  }, []);

  // ... openWidget, handleOpenAll, handleOpenSingle — see step 2
}

Add these helpers to widgetConfig.ts alongside the npm exports:

  'https://cdn.jsdelivr.net/npm/reimaginehome-widget/dist/widget.js';

export function buildScriptEmbedWidgetConfig() {
  return {
    media: resolveListingMedia(),
    mode: 'simple' as const,
    branding: buildWidgetBranding(),
    sidebar_position: 'right' as const,
    language: buildWidgetLanguage(),
    onComplete: (detail) => console.log('[reih] onComplete:', detail),
    onError: (err) => console.error('[reih] onError:', err),
    onClose: () => console.log('[reih] onClose'),
  };
}

export function waitForReihWidget(timeoutMs = 30_000) {
  if (window.reihWidget?.open) return Promise.resolve(window.reihWidget);

  return new Promise((resolve, reject) => {
    const poller = setInterval(() => {
      if (window.reihWidget?.open) {
        clearInterval(poller);
        resolve(window.reihWidget);
      }
    }, 50);
    setTimeout(() => { clearInterval(poller); reject(new Error('Widget script did not load')); }, timeoutMs);
  });
}

export function createWidgetOpener(getWidget, logPrefix) {

  return function openWidget(media) {
    if (opening) return;
    opening = true;
    Promise.resolve(getWidget())
      .then((widget) => openReihWithMedia(widget, media))
      .catch((err) => { clearReihLoader(); console.error(logPrefix, err); })
      .finally(() => { opening = false; });
  };
}

1. Set config and inject the script

script-embed.js
import {
  WIDGET_PUBLIC_KEY,
  applyHostCssVars,
  buildScriptEmbedWidgetConfig,
  clearReihLoader,
  createWidgetOpener,
  initListingPage,
} from "./shared.js";

var WIDGET_SCRIPT_URL =
  "https://cdn.jsdelivr.net/npm/reimaginehome-widget/dist/widget.js";
var WIDGET_SCRIPT_ID = "reih-widget-script";

function waitForReihWidget(timeoutMs) {
  if (window.reihWidget?.open) return Promise.resolve(window.reihWidget);
  return new Promise(function (resolve, reject) {
    var poller = setInterval(function () {
      if (window.reihWidget?.open) { clearInterval(poller); resolve(window.reihWidget); }
    }, 50);
    setTimeout(function () { clearInterval(poller); reject(new Error("Widget script did not load")); }, timeoutMs || 30000);
  });
}

function loadWidgetScript() {
  if (document.getElementById(WIDGET_SCRIPT_ID)) return;
  var script = document.createElement("script");
  script.id = WIDGET_SCRIPT_ID;
  script.src = WIDGET_SCRIPT_URL + "?v=" + Date.now();
  script.async = true;
  script.setAttribute("data-public-key", WIDGET_PUBLIC_KEY);
  document.body.appendChild(script);
}

var openWidget = createWidgetOpener(waitForReihWidget, "[script-embed]");

3. Route-scoped page component

Widget lifecycle lives in a standalone page component — not a global root service. Configure in ngOnInit, destroy in ngOnDestroy.

src/app/pages/package-page.component.ts
import { isPlatformBrowser } from '@angular/common';
import { Component, OnDestroy, OnInit, PLATFORM_ID, inject } from '@angular/core';
import { reihWidget } from 'reimaginehome-widget';
import { ListingDemoPageComponent } from '../components/listing-demo-page.component';
import {
  buildNpmWidgetConfigureOptions,
  clearReihLoader,
  openReihWithMedia,
  resolveListingMedia,
  type ReihMediaItem,
} from '../widget/widget-config';

@Component({
  selector: 'app-package-page',
  imports: [ListingDemoPageComponent],
  template: `
    <app-listing-demo-page
      (openAll)="handleOpenAll()"
      (openSingle)="handleOpenSingle($event)"
    />
  `,
})
export class PackagePageComponent implements OnInit, OnDestroy {
  private readonly platformId = inject(PLATFORM_ID);
  private opening = false;

  ngOnInit(): void {
    if (!isPlatformBrowser(this.platformId)) return;
    reihWidget.configure(buildNpmWidgetConfigureOptions());
  }

  ngOnDestroy(): void {
    if (!isPlatformBrowser(this.platformId)) return;
    reihWidget.destroy();
    clearReihLoader();
  }

  private async openWidget(media: ReihMediaItem[]): Promise<void> {
    if (this.opening) return;
    this.opening = true;
    try {
      await openReihWithMedia(reihWidget, media);
    } finally {
      this.opening = false;
    }
  }

  handleOpenAll(): void {
    void this.openWidget(resolveListingMedia());
  }

  handleOpenSingle(media: ReihMediaItem): void {
    void this.openWidget([media]);
  }
}

Styling & branding

The widget renders inside its own overlay and isn't styleable by your page CSS — but it accepts a constrained set of branding tokens so it can match your visual identity.

What you can customize

  • Primary color — used for buttons, highlights, and focus states. Pass any hex value that meets WCAG AA contrast with white text.
  • Heading text — the headline at the top of the widget. Defaults to "Reimagine this space".
  • Co-brand logo — shown alongside the ReimagineHome mark in the header. SVG preferred, PNG accepted.
  • Language — initial language selection. Users can switch inside the widget.

What you cannot customize

  • The widget's layout, typography, or component structure.
  • The ReimagineHome attribution mark.
  • The style-choice UI generated from room-type detection.
