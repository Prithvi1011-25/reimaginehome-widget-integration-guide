Turn listing photos into visual decisions inside your platform.

Install & usage

Request a publishable key from partners@styldod.com, then follow the install and usage steps below. Pick CDN for a script-tag embed, or npm for React, Next.js, Vue, or Angular.

Download & install

npm

Install the package from npm:

$ npm install reimaginehome-widget

CDN

Or load from jsDelivr CDN. Set your publishable key on the script tag — not inside reihWidgetConfig:

<script async
  src="https://cdn.jsdelivr.net/npm/reimaginehome-widget/dist/widget.js"
  data-public-key="public_key"></script>

Usage

npm integration

1. Import the widget:

import { reihWidget } from 'reimaginehome-widget';

2. Configure once on page load:

Pass your publishable key and any persistent defaults. The widget keeps this until you reconfigure or destroy it.

reihWidget.configure({
  public_key: 'public_key',
  mode: 'simple',
  sidebar_position: 'right',
  onError: (err) => console.error(err),
  onClose: () => console.log('widget closed'),
});

3. Open on user action:

Call reihWidget.open() when the user clicks your trigger. Pass publicly reachable image URLs in media.

await reihWidget.open({
  media: [
    { image_url: 'https://yoursite.com/listings/01.webp' },
    { image_url: 'https://yoursite.com/listings/02.webp' },
  ],
});

4. Close or destroy when done:

Use close() to hide the overlay while keeping the session warm. Use destroy() on route change in SPAs.

reihWidget.close();    // hide modal, keep session warm
reihWidget.destroy();  // tear down completely

CDN integration

1. Set config before opening:

Assign window.reihWidgetConfig with media and options. The publishable key stays on data-public-key.

window.reihWidgetConfig = {
  media: [
    { image_url: 'https://yoursite.com/listings/01.webp' },
  ],
  mode: 'simple',
  sidebar_position: 'right',
};

2. Open after the script loads:

Wait for window.reihWidget, then call open(). You can pass per-launch overrides here.

await window.reihWidget.open({
  media: [
    { image_url: 'https://yoursite.com/listings/01.webp' },
  ],
});

3. Clean up on page leave:

window.reihWidget?.destroy();

Configuration reference

Persistent configuration applied to every open() call until you reconfigure or destroy. Set via reihWidget.configure() (npm) or window.reihWidgetConfig + data-public-key (CDN).

CDN exception. For CDN integrations, public_key is set on the script tag's data-public-key attribute, not inside window.reihWidgetConfig. All other fields below go in the config object.

Configuration params

Argument (Default value)
Description
#public_key required · string

Your publishable key. Set as public_key in configure options or data-public-key on the script tag. Bound to your allowed origins.

#mode "simple"

Widget flow mode. One of "simple" or "agentic". The "agentic" mode is available; documentation forthcoming.

#branding undefined

Logo, heading, colors, footer. See Styling & branding for the sub-fields.

#theme undefined

Colors, font, radius — visual tokens applied to the widget UI.

#language undefined 
language: [
  { code: "en-US", name: "English (United States)", nativeName: "English (US)" },
  { code: "en-GB", name: "English (United Kingdom)", nativeName: "English (UK)" },
  { code: "es-ES", name: "Spanish", nativeName: "Español" },
]

Languages available in the in-widget switcher. Each entry is a WidgetLanguage object with code, name, and nativeName.

#sidebar_position undefined

Which side of the overlay the sidebar mounts on. One of "left" or "right".

#user_id undefined

Your own analytics identifier for the end user. Echoed in events.

#session_id undefined

Your own reference id for this user's flow. Distinct from the widget's internal session.

reihWidget.open() method

reihWidget.open(overrides?) is async and returns a Promise. The optional overrides argument is per-launch — media is required, everything else overrides the persistent config from configure() for this launch only.

reihWidget.open() overrides

Argument (Default value)
Description
#media required · MediaItem[] 
media: [
  { image_url: "https://yoursite.com/listings/01.webp" },
  { image_url: "https://yoursite.com/listings/02.webp" },
]

Non-empty array. Each item is a MediaItem with at minimum an image_url pointing to a publicly reachable image. Pass one for single-media flow, many for batch.

#mode inherits configure()

Override the configured mode for this launch. One of "simple" or "agentic".

#branding inherits configure()

Override the persistent branding for this launch only. Useful for multi-brand portals.

#theme inherits configure()

Override the persistent theme for this launch.

#language inherits configure()

Override the available languages for this launch.

#sidebar_position inherits configure()

Override the sidebar position for this launch. One of "left" or "right".

#onError undefined

Callback invoked with { message, code? } when something goes wrong during the session.

#onClose undefined

Callback invoked when the modal closes.

Session lifecycle

Two helpers cover the session after reihWidget.open():

reihWidget.close();    // hide modal, keep session warm
reihWidget.destroy();  // tear down — use on SPA route change

Styling & branding

The widget renders inside its own overlay and isn't styleable by your page CSS — but it accepts a constrained set of branding tokens so it can match your visual identity.

What you can customize

  • Primary color — used for buttons, highlights, and focus states. Pass any hex value that meets WCAG AA contrast with white text.
  • Heading text — the headline at the top of the widget. Defaults to "Reimagine this space".
  • Co-brand logo — shown alongside the ReimagineHome mark in the header. SVG preferred, PNG accepted.
  • Language — initial language selection. Users can switch inside the widget.

What you cannot customize

  • The widget's layout, typography, or component structure.
  • The ReimagineHome attribution mark.
  • The style-choice UI generated from room-type detection.

Example

reihWidget.configure({
  public_key: 'public_key',
  branding: {
    logo: 'https://yoursite.com/logo.svg',
    colors: {
      primary: '#1F6F4A',
      text_primary: '#071121',
    },
  },
  language: [
    { code: 'en-US', name: 'English (United States)', nativeName: 'English (US)' },
  ],
});
